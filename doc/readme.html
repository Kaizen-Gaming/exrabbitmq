    ExRabbitMQ

    RabbitMQ client library, written in Elixir, which abstracts away connection and channel lifecycle handling.

    It provides hooks and default function implementations, to allow library users to consume and publish messages, minimizing the need to directly interact with the AMQP module.

    Configuration

    You can provide configuration for connections and queues in two ways.

    1. Define configuration sections under application :exrabbitmq with appropriate atom keys and pass these keys to c:ExRabbitMQ.Consumer.xrmq_init/2 and c:ExRabbitMQ.Producer.xrmq_init/1.
    2. Use the configuration structs ExRabbitMQ.ConnectionConfig and ExRabbitMQ.Consumer.QueueConfig and pass them directly to c:ExRabbitMQ.Consumer.xrmq_init/2 and c:ExRabbitMQ.Producer.xrmq_init/1.

    Example usage

    For an example usage please refer to test/ex_rabbit_m_q_test.exs.

    There you will find a complete producer/consumer example, which you can run given a local RabbitMQ installation at 127.0.0.1:5672 with the default credentials guest:guest.
